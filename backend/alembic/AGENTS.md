# alembic

## Purpose
Database migrations for the backend. `env.py` wires Alembic to the SQLAlchemy metadata in `../db/`; `versions/` holds autogenerated migration scripts.

## Key Files
| File | Description |
|------|-------------|
| `env.py` | Alembic environment — connects Alembic to `../db/base.py` metadata |

## Subdirectories
| Directory | Purpose |
|-----------|---------|
| `versions/` | Generated migration scripts (one per schema change) — not individually documented |

## For AI Agents

### Working In This Directory
- Schema changes trigger cross-model review (per `CLAUDE.md`).
- Generate: `cd backend && alembic revision --autogenerate -m "message"`.
- Review every autogenerated migration — Alembic misses type changes and indexes sometimes.
- Never edit a migration that has already been applied in production; write a new one.

### Testing Requirements
- `../tests/test_migrations.py` verifies apply + rollback against an empty DB.

## Dependencies

### Internal
- `../db/base.py` (metadata), `../db/models.py`, `../db/batch_models.py`.

### External
- Alembic.
